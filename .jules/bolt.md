## 2024-04-17 - [Optimizing Component Render Cycle]
**Learning:** React re-renders component on state change. Static data arrays instantiated within a component are recreated on every render. Complex mapping or data derivation logic tied to controlled inputs like text fields will run repeatedly as the user types unless memoized with useMemo. Repeating string transformations (like `.toLowerCase()`) inside filter loops adds unnecessary overhead.
**Action:** Always move static configuration arrays outside of functional components or use `useMemo`. Memoize computationally heavy derived state (like filtering lists) using `useMemo` when dependent on frequent state updates (e.g., text inputs). Hoist loop-invariant calculations (like converting a search term to lowercase) outside the iterative operation.
